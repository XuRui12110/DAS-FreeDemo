Git & GitHub Setup and Practice Checklist

1. Setup

  • Create an account at GitHub.com
  • Install git on your machine
  • Install GitHub Desktop
  • Log into your GitHub account inside GitHub Desktop

2. Create Your First Repository - using GitHub Desktop

  • Create a blank repository
    • Add a file to your repo
    • Stage your file (this is automatic in GitHub Desktop)
    • Commit your changes with a commit message
    • Push/Publish your updated repo to GitHub server
    • Time to try something scary...
      • Delete your local folder from your machine! Scary!
      • Clone your repo onto your machine from the GitHub server

3. Working with Files

  • Create at least one new file in your repo
  • Modify an existing file in your repo
  • Commit the changes, and push to GitHub server

4. Exploring on GitHub.com

  • Visit your repo at the GitHub.com website
    • Look at the history of commits and commit messages
    • Make a small edit to a file on the GitHub website
    • Commit your small change to your repo, with a commit message
    • Return to GitHub Desktop, fetch and pull the repo from the server
    • Inspect the file you edited on the server, to see if the change has arrived

5. Advanced Experiment

Want to try and break things? Yes please!

Warning: These are step not recommended normally; they are just to show you something to avoid.

  • Make a file change in your local repo on your machine
  • Commit your changes but DO NOT push to the GitHub server
  • Return to GitHub Desktop and try to push your repo to the GitHub server
  • Visit the GitHub website and change the same file (in a different way) on the website
  • Commit your change on GitHub.com
